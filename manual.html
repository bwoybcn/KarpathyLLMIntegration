01What Is This?

A system that turns scattered sources into structured, interconnected knowledge — automatically.

Imagine you're researching a topic. You find a dozen articles, a few papers, some GitHub repos. Normally you'd read them, take some notes, and eventually forget half of what you learned.

This system does something different. You point it at your sources — or tell it to find sources — and it:

  1. Ingests them into a structured folder
  2. Compiles them into a wiki of interconnected articles, each with cross-references (wikilinks)
  3. Answers questions by researching across the wiki and citing its sources
  4. Finds gaps in your knowledge and suggests (or fetches) sources to fill them
  5. Grows over time — your queries and explorations feed back into the wiki

Everything lives in an Obsidian vault — a folder of markdown files you can browse, search, and visualize with Obsidian's graph view.

Key Idea

The AI writes and maintains the wiki. You rarely edit it directly. Instead, you ingest sources, ask questions, and let the system build your knowledge base.

What it works for

Use CaseExample

02Quick Start

From zero to a working knowledge base in five steps.

  1. Create a vault

    Open Claude Code and type:

    /kb-new "Transformer Architecture"

    This creates a pre-configured Obsidian vault at D:\KnowledgeBases\transformer-architecture\.

  2. Let the AI find sources 
    /kb-research "transformer architecture and attention mechanisms"

    The system searches the web, evaluates candidates for quality, and ingests the best 3-8 sources into your vault. This takes 1-3 minutes.

  3. Compile the wiki 
    /kb-compile

    Each source is processed one at a time. The AI extracts concepts, creates articles with [[wikilinks]], and builds an index.

  4. Ask a question 
    /kb-query "How does multi-head attention work?"

    The AI reads relevant wiki articles and gives you an answer with citations.

  5. Open in Obsidian

    Open Obsidian, click "Open folder as vault", and select your vault folder. Enable the community plugins when prompted. Explore the graph view to see how concepts connect.

That's it.

You now have a structured knowledge base. Run /kb-expand to automatically find and fill gaps, or /kb-research again with different subtopics to grow it further.

Each stage explained

Research & Ingest — Sources enter the system. Either the AI finds them autonomously (/kb-research) or you point it at specific URLs/files (/kb-ingest). Sources are saved as markdown in raw/.

Compile — The AI reads each source, extracts key concepts, and writes structured wiki articles. If a concept already exists, it merges new information without deleting anything. Every claim is attributed to its source.

Query — Ask questions against the wiki. The AI reads the index, identifies relevant articles, and synthesizes answers with [[wikilink]] citations. Answers are automatically saved to outputs/reports/ so every query enriches the vault.

Lint — Health checks find broken links, missing cross-references, contradictions, and gaps. The system suggests what to research next.

Expand — Reads the lint report and autonomously researches sources to fill the gaps. The wiki grows itself.

The Karpathy Principle

"Your own explorations and queries always add up in the knowledge base." Every question you ask and every answer you get can be filed back, making the wiki richer for next time.

04Command Reference

All commands are typed in Claude Code's prompt.

/kb-new {topic}
Create a new knowledge base vault with pre-configured Obsidian settings.
/kb-new "Machine Learning Fundamentals"
/kb-research {topic}
Autonomously search the web, evaluate source quality, and ingest the best ones. Supports --type (any, academic, web, repos, news) and --date_range.
/kb-research "attention mechanisms" --type academic
/kb-ingest {url | path}
Manually ingest a specific source — a URL, PDF, arXiv paper, GitHub repo, or markdown file.
/kb-ingest https://arxiv.org/abs/1706.03762
/kb-compile
Compile new or modified raw sources into wiki articles with wikilinks and concept extraction.
/kb-compile
/kb-query {question}
Ask a question against the wiki. The AI researches the answer across articles and cites sources.
/kb-query "What are the trade-offs between CNN and transformer architectures?"
/kb-lint
Run health checks: broken links, missing cross-references, inconsistencies, and data gap suggestions.
/kb-lint
/kb-expand
Auto-identify knowledge gaps and research sources to fill them. Combines lint → research → compile.
/kb-expand --max_sources 5
/kb-status
Show vault statistics: article count, word count, uncompiled sources, broken links.
/kb-status
/kb-output {format}
Generate outputs from wiki content. Formats: slides (Marp), report, chart, summary.
/kb-output slides "Overview of Attention Mechanisms"

05Vault Structure

Each knowledge base is a folder of markdown files — simple, portable, and human-readable.

D:\KnowledgeBases\your-topic\ ├── raw/ ← Source documents go here │ ├── web/ ← Web articles │ ├── papers/ ← PDFs, arXiv papers │ ├── repos/ ← GitHub repo docs │ └── notes/ ← Your own notes │ ├── wiki/ ← AI-compiled knowledge │ ├── concepts/ ← One article per concept │ ├── sources/ ← Summary of each source │ ├── categories/ ← Topic groupings │ ├── _index.md ← Master article list │ ├── _categories.md ← Category tree │ ├── _glossary.md ← Term definitions │ └── _backlinks.md ← Reverse link index │ ├── outputs/ ← Generated deliverables │ ├── reports/ │ ├── slides/ ← Marp slide decks │ └── charts/ ← Matplotlib images │ ├── _meta/ ← System state │ ├── compilation-state.json │ ├── stats.md │ └── lint-report.md │ ├── AGENTS.md ← Rules for AI behavior └── .obsidian/ ← Obsidian config

How articles are structured

Every wiki article follows a consistent format with YAML frontmatter (metadata at the top) and structured sections:

---
title: "Attention Mechanism"
type: concept
sources: ["raw/papers/vaswani-2017.md"]
category: "[[Neural Networks]]"
confidence: high
---

# Attention Mechanism

Summary paragraph.

## Overview
Explanation with [[wikilinks]] to related concepts.

## Evidence and Sources
- From [[Source: Vaswani 2017]]: specific finding

## Related Concepts
- [[Transformer Architecture]] — builds on this
- [[Self-Attention]] — a specific variant

The [[wikilinks]] are what make everything connected. Click them in Obsidian to navigate between concepts, or view the graph to see the full network.

06News & General Topics

The system isn't just for academic research — it handles news stories, current events, and general topics too.

Date-filtered news research

/kb-research "UK general election" --type news --date_range "2025-06-01 to 2025-07-31"

When you use --type news, the system changes its behavior:

  • Different search strategy — targets wire services (Reuters, AP), major outlets (BBC, Guardian, NYT), analysis pieces, and alternative perspectives
  • Multi-perspective requirement — ingests at least 2 different outlets per major event to avoid single-source bias
  • Date discipline — skips articles outside the requested date range
  • Event deduplication — if 5 articles cover the same press conference, it picks the best 2-3 from different outlets
  • Timeline articles — creates chronological event sequences instead of static concept articles

Timeline articles

For news topics, the compiler creates timeline articles that show events in chronological order with multiple perspectives:

---
title: "Timeline: UK Election Campaign"
type: timeline
date_start: 2025-06-01
date_end: 2025-07-04
---

# Timeline: UK Election Campaign

## 2025-06-12 — Manifesto Launches
- From [[Source: BBC]]: Labour pledges...
- From [[Source: Guardian]]: Conservative response...

## 2025-06-18 — Leaders' Debate
- From [[Source: Reuters]]: Key moments...

## Conflicting Accounts
- Source A reports X, Source B reports Y

Source quality tiers for news

TierExamplesTreatment
Tier 1Reuters, AP, BBC, NPRAlways ingested — factual baseline
Tier 2Guardian, NYT, FT, specialist pressUsually ingested — depth and analysis
Tier 3Regional outlets, opinion columnsSelective — only if unique perspective
Tier 4Tabloids, aggregators, AI summariesSkipped

07Tips & Patterns

Get the most out of the system with these workflows.

The "hands-off" workflow

/kb-new "Your Topic"
/kb-research "your topic" --max_sources 8
/kb-compile
/kb-expand
/kb-status

Four commands and you have a structured wiki. Run /kb-expand again periodically to keep it growing.

File query answers back

After /kb-query, you can ask to save the answer into the wiki. This means your own investigations make the knowledge base richer — exactly Karpathy's "explorations add up" principle.

Open the Dataview dashboard

After compiling, open wiki/_dashboards.md in Obsidian. It contains live Dataview queries that update automatically:

  • Articles grouped by type (concept, source, timeline)
  • High and low confidence articles
  • Recently updated articles
  • Articles with fewest sources (need more evidence)
  • Source summaries by type
  • Full tag distribution

Use the concept map canvas

Open wiki/_concept-map.canvas for a spatial, zoomable view of how concepts relate. Each concept is a node, each wikilink is an edge. You can drag nodes to rearrange the layout.

Use the Obsidian graph view

The vault is pre-configured with color-coded graph view:

  • Blue — concept articles
  • Green — source summaries
  • Orange — categories
  • Gray — raw sources
  • Purple — outputs

Callouts make articles scannable

Articles use Obsidian's callout syntax for visual scanning:

  • Abstract (blue) — article summary at the top
  • Example (green) — key findings and data points
  • Question (orange) — open questions and uncertainties
  • Warning (red) — contradictions between sources
  • Quote — notable quotes from sources

Structured tags for filtering

Tags use a hierarchical format visible in Obsidian's tag pane:

  • #type/concept, #type/source, #type/timeline
  • #confidence/high, #confidence/medium, #confidence/low
  • #source-type/paper, #source-type/web, #source-type/news
  • #topic/your-domain — custom topic tags

Multiple vaults for different topics

Each /kb-new creates a separate Obsidian vault. Keep topics isolated — a vault on "Machine Learning" and another on "Climate Policy" work independently.

Ingest from different source types

You give itWhat happens

08Installation

What you need and how to set it up.

Requirements

SoftwarePurposeInstall
Claude CodeThe AI engineclaude.ai/claude-code
ObsidianWiki viewerobsidian.md
Python 3.10+Backend scriptpython.org
Node.js 18+Hook scriptsnodejs.org
Defuddle CLIWeb article extractionnpm install -g defuddle-cli

Install steps

  1. Clone the repository: git clone https://github.com/bwoybcn/KarpathyLLMIntegration.git
  2. Copy files from source-files/ into your Claude Code home (~/.claude/)
  3. Register the auto-compile hook in ~/.claude/settings.json
  4. Optionally configure MarkItDown MCP (for PDFs) and Zotero MCP (for papers)
  5. Set the KB_ROOT environment variable if you want vaults somewhere other than D:\KnowledgeBases\
Full instructions

See INSTALL.md in the repository for detailed, step-by-step installation with platform-specific guidance for Windows, macOS, and Linux.

09Troubleshooting

ProblemSolution
"Not inside a KB vault"Either cd into a vault directory or specify --vault name
Compilation seems slowSources are processed one at a time for quality. Large papers take longer.
Broken wikilinks after compileRun /kb-lint --fix true to auto-create stub articles
Dataview queries show as raw codeUpdate Dataview: Settings → Community Plugins → Check for updates. Then switch to Reading view (Ctrl+E).
Obsidian shows no pluginsGo to Settings → Community Plugins → Turn on, then install Dataview and Marp Slides
MarkItDown not workingCheck it's configured in ~/.claude/settings.json under mcpServers
defuddle command not foundRun npm install -g defuddle-cli
