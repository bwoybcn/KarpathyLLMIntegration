Setup

/kb-new
CREATE Create a new knowledge base vault

Creates a pre-configured Obsidian vault with the full directory structure, graph view colors, Dataview dashboards, and empty indexes. Each topic gets its own vault.

/kb-new "Transformer Architecture"
ArgumentDescription
topicName for the knowledge base. Gets slugified for the filesystem.
Ingestion
🔍
/kb-research
RESEARCH Autonomously find and ingest sources

Searches the web for high-quality sources on a topic, evaluates them for quality and relevance, and ingests the best ones into the vault. Supports academic, web, repo, and news modes with optional date filtering.

/kb-research "attention mechanisms" --type academic
/kb-research "UK election" --type news --date_range "2025-06-01 to 2025-07-31"
ArgumentDescription
topicWhat to research.
--type optionalany | academic | web | repos | news. Default: any.
--date_range optionalDate filter, e.g. "2025-01-01 to 2025-12-31" or "last 30 days".
--max_sources optionalMax sources to ingest. Default: 5.
--vault optionalVault name or path. Auto-detected from CWD.
📥
/kb-ingest
INGEST Manually add a specific source

Ingest a specific URL, file, or paper into the vault. Automatically detects the source type and uses the right extraction tool: defuddle for web, AlphaXiv for arXiv papers, MarkItDown for PDFs, Zotero for academic papers.

/kb-ingest https://arxiv.org/abs/1706.03762
/kb-ingest D:\papers\my-paper.pdf
ArgumentDescription
sourceURL, file path, or Zotero item key.
--vault optionalVault name or path. Auto-detected from CWD.
Compilation
⚙
/kb-compile
COMPILE Build the wiki from raw sources

The core engine. Reads each new or modified source, extracts concepts, creates wiki articles with [[wikilinks]], and merges new information into existing articles. Rebuilds all indexes, validates links, and updates the concept map canvas.

What happens when you compile

  1. Diff detection — finds new or modified files in raw/
  2. Per-source processing — extracts concepts, creates source summaries, creates or merges concept articles
  3. Index rebuild — regenerates _index.md, _categories.md, _glossary.md, _backlinks.md
  4. Link validation — checks all [[wikilinks]] resolve correctly
  5. Canvas update — regenerates the concept relationship map
  6. Stats update — recalculates article count, word count, health metrics
/kb-compile
/kb-compile --source raw/web/specific-article.md
ArgumentDescription
--source optionalCompile only this specific raw file. Compiles all new sources if omitted.
--vault optionalVault name or path. Auto-detected from CWD.
Query & Output
?
/kb-query
QUERY Ask questions against the wiki

Researches your question by reading the wiki index, identifying relevant articles, and synthesizing an answer with [[wikilink]] citations. Answers are automatically saved to outputs/reports/ so every query enriches the vault.

/kb-query "How does multi-head attention differ from single-head?"
ArgumentDescription
questionThe question to research.
--vault optionalVault name or path.
📊
/kb-output
OUTPUT Generate deliverables from the wiki

Generates formatted outputs from wiki content. Slide decks use Marp format (viewable in Obsidian with the Marp plugin), charts use matplotlib, and reports are long-form markdown with citations.

/kb-output slides "Overview of Attention Mechanisms"
/kb-output report "Comparison of TNE delivery models"
ArgumentDescription
formatslides | report | chart | summary
topic optionalWhat to generate output about.
--vault optionalVault name or path.
Maintenance
🔎
/kb-lint
MAINTAIN Health checks and quality audit

Runs structural checks (broken wikilinks, orphan articles, frontmatter validation) and semantic checks (inconsistencies, missing cross-references, coverage gaps). Writes a lint report to _meta/lint-report.md viewable in Obsidian.

/kb-lint
/kb-lint --fix true
ArgumentDescription
--fix optionalAuto-fix structural issues (create stubs for broken links, fix frontmatter). Default: false.
--vault optionalVault name or path.
🌱
/kb-expand
GROW Auto-fill knowledge gaps

The autonomous growth engine. Reads the lint report's gaps and suggestions, researches sources to fill them, compiles the results, and reports what grew. Combines /kb-lint → /kb-research → /kb-compile in one command. Can target a specific topic or let the system choose based on gaps.

/kb-expand
/kb-expand "China as a market for Russell Group"
ArgumentDescription
--max_sources optionalTotal source budget across all gaps. Default: 10.
--vault optionalVault name or path.
📋
/kb-status
STATUS Dashboard and health summary

Shows a quick dashboard of the vault's current state: article count, word count, source count, categories, uncompiled sources, and broken links. Suggests next actions based on the health metrics.

/kb-status
/kb-status --vault transnational-education
Automatic Logging

Every operation leaves a trace

All logs are append-only markdown files in _meta/, viewable in Obsidian. The vault records not just what you know, but how you built that knowledge.

/kb-research_meta/research-log.mdSearches run, sources ingested, sources skipped with reasons
/kb-compile_meta/compilation-log.mdSources processed, articles created and merged, wiki stats
/kb-lint_meta/lint-history.mdIssue counts over time (critical, warnings, suggestions)
/kb-expand_meta/expansion-log.mdGaps targeted, before/after article and word counts
/kb-status_meta/stats-history.mdTimestamped snapshot: articles, words, sources, broken links

Typical Workflow

From zero to a living knowledge base

  1. /kb-new "Your Topic" — create the vault
  2. /kb-research "your topic" — AI finds and ingests 5 sources
  3. /kb-compile — builds the wiki with wikilinks and callouts
  4. /kb-query "your question" — get answers with citations
  5. /kb-lint — find gaps and issues
  6. /kb-expand — auto-fill the gaps, wiki grows
  7. Repeat 4–6 — the wiki gets richer with every cycle
